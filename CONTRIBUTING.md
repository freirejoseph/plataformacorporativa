# Contributing

## Scope
- Work inside `D:/PROYECTOS/PLATAFORMACORPORATIVA` for the local mirror.
- Keep `/home/plataformacorporativa` in sync through the sync routine.

## Rules
- Do not commit secrets from `.env`.
- Keep changes focused and small.
- Update the relevant `SKILL.md` when a module changes.
- Add or update tests when behavior changes.
- Prefer ASCII in new files unless a file already uses accents.

## Workflow
1. Read the module `SKILL.md`.
2. Make the change.
3. Run the local validation step.
4. Sync to Ubuntu if the change belongs in the remote workspace.
5. Update the QA review file if the change resolves a reported item.
